Lists【列表】 and Keys

作者： Lian

日期： 2018年05月23日 20:05

First, let’s review【回顾】 how you transform【转换】 lists in JavaScript.

Given the code below【下面】, we use the map() function to take an array【数组】 of numbers and double【两倍】 their values. We assign【分配】 the new array returned by map() to the variable【变量】 doubled and log it:

const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map((number) => number * 2);
console.log(doubled);

This code【代码】 logs [2, 4, 6, 8, 10] to the console.

In React, transforming【转换】 arrays into lists of elements is nearly【几乎】 identical.

    Keys【钥匙】

    Keys help【帮助】 React identify【识别】 which items have changed【改变】, are added【增加】, or are removed【删除】. Keys should be given to the elements inside the array to give the elements a stable identity【识别】:

    const numbers = [1, 2, 3, 4, 5];
const listItems = numbers.map((number) =>
  
  • 
    {number}
  
    • 
);

    The best way to pick a key is to use【使用】 a string that uniquely【唯一】 identifies a list item among its siblings. Most often you would use IDs from your data as keys:

    const todoItems = todos.map((todo) =>
  
  • 
    {todo.text}
  
    • 
);

    When you don’t have stable【稳定】 IDs for rendered【渲染】 items, you may use the item【个体】 index as a key as a last resort:

    const todoItems = todos.map((todo, index) =>
  // Only do this if items have no stable IDs
  
  • 
    {todo.text}
  
    • 
);

    We don’t recommend using indexes for keys if the order of items may change. This can negatively impact performance and may cause issues with component state. Check out Robin Pokorny’s article for an in-depth explanation on the negative impacts of using an index as a key. If you choose not to assign an explicit key to list items then React will default to using indexes as keys.

    Here is an in-depth explanation about why keys are necessary if you’re interested in learning more.

    Extracting Components with Keys

    Keys only make sense in the context of the surrounding array.

    For example, if you extract a ListItem component, you should keep the key on the <ListItem /> elements in the array rather than on the <li> element in the ListItem itself.
