introducing【介绍】 jsx

作者： Lian

日期： 2018年05月23日 17:05

Consider this variable【变量】 declaration【声明】:

const element = <h1>Hello, world!</h1>;

This funny tag【标签】 syntax【语法】 is neither a string nor HTML.

It is called JSX, and it is a syntax extension【扩展】 to JavaScript. We recommend【推荐】 using it with React to describe【描述】 what the UI should look【看起来】 like. JSX may remind【提醒】 you of a template【模板】 language, but it comes with【拥有】 the full power【能力】 of JavaScript.

JSX produces【生成】 React “elements”. We will explore【导出】 rendering【渲染】 them to the DOM in the next section. Below, you can find the basics【基本】 of JSX necessary【必要】 to get you started.

Why【为什么】 JSX?

React embraces【包含】 the fact that rendering logic【逻辑】 is inherently【继承】 coupled with other UI logic: how events【事件】 are handled【处理】, how the state【状态】 changes【变化】 over time, and how the data is prepared【准备】 for display【显示】.

Instead of artificially separating【分离】 technologies by putting markup and logic in separate files, React separates concerns with loosely coupled【松耦合】 units【单元】 called “components”【组件】 that contain both【两者】. We will come back to components in a further section, but if you’re not yet comfortable【舒适】 putting markup in JS, this talk might convince【说服】 you otherwise.

React doesn’t require using【使用】 JSX, but【但是】 most people find it helpful【有用】 as a visual【虚拟】 aid【帮助】 when working with UI inside【内部】 the JavaScript code. It also allows React to show more useful error【错误】 and warning【警告】 messages.

With that out of the way, let’s get started【开始】!

Embedding【嵌入】 Expressions【表达式】 in JSX

In the example【示例】 below, we declare【声明】 a variable called name and then use it inside JSX by wrapping【包裹】 it in curly braces:

const name = 'Josh Perez';
const element = 
Hello, {name}
;

ReactDOM.render(
  element,
  document.getElementById('root')
);

You can put any valid【有效】 JavaScript expression inside the curly braces【大括号】 in JSX. For example, 2 + 2, user.firstName, or formatName(user) are all valid JavaScript expressions.

In the example below, we embed【嵌入】 the result【结果】 of calling【调用】 a JavaScript function【函数】, formatName(user), into an <h1> element.

function formatName(user) {
  return user.firstName + ' ' + user.lastName;
}

const user = {
  firstName: 'Harper',
  lastName: 'Perez'
};

const element = (
  

    Hello, {formatName(user)}!
  

);

ReactDOM.render(
  element,
  document.getElementById('root')
);

.

We split【分离】 JSX over multiple【多】 lines for readability【可读性】. While it isn’t required, when doing this, we also recommend wrapping【包裹】 it in parentheses【圆括号】 to avoid the pitfalls【陷阱】 of automatic semicolon insertion.

JSX is an Expression【表达式】 Too

After compilation【汇编】, JSX expressions become regular【常规】 JavaScript function calls【调用】 and evaluate【求值】 to JavaScript objects.

This means that you can use JSX inside of if statements【条件语句】 and for loops【循环】, assign【分配】 it to variables【变量】, accept it as arguments【参数】, and return【返回】 it from functions:

function getGreeting(user) {
  if (user) {
    return 
Hello, {formatName(user)}!
;
  }
  return 
Hello, Stranger.
;
}

Specifying【指定】 Attributes【属性】 with JSX

You may use quotes【引号】 to specify【指定】 string literals【字面量】 as attributes:

const element = <div tabIndex="0"></div>;

You may also use curly braces【大括号】 to embed a JavaScript expression【表达式】 in an attribute:

const element = <img src={user.avatarUrl}></img>;

Don’t put quotes【引号】 around curly braces【大括号】 when embedding【嵌入】 a JavaScript expression【表达式】 in an attribute【属性】. You should either use quotes (for string values) or curly braces (for expressions), but not both【两者】 in the same attribute.

Warning:

Since JSX is closer【更近】 to JavaScript than to HTML, React DOM uses camelCase property【合理】 naming convention【习俗】 instead of HTML attribute names.

For example【示例】, class becomes className in JSX, and tabindex becomes tabIndex.

Specifying【指定】 Children【子块】 with JSX

If a tag【标签】 is empty【空】, you may close【关闭】 it immediately【立刻】 with />, like XML:

const element = <img src={user.avatarUrl} />;

JSX tags may contain【包含】 children:

const element = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>
);

JSX Prevents【阻止】 Injection【注入】 Attacks【攻击】

It is safe【安全】 to embed user input in JSX:

const title = response.potentiallyMaliciousInput;
// This is safe:
const element = <h1>{title}</h1>;

By default【默认】, React DOM escapes any values【数据】 embedded【嵌入】 in JSX before rendering【渲染】 them. Thus it ensures【保证】 that you can never inject anything that’s not explicitly【明确】 written in your application. Everything is converted【转换】 to a string before being rendered【渲染】. This helps prevent XSS (cross-site-scripting) attacks.

JSX Represents【表示】 Objects【对象】

Babel compiles【编译】 JSX down to React.createElement() calls.

These two examples are identical:

const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

const element = React.createElement(
  'h1',
  {className: 'greeting'},
  'Hello, world!'
);

React.createElement() performs a few checks【检查】 to help you write bug-free【更少bug】 code but essentially it creates【创建】 an object like this:

// Note: this structure is simplified
const element = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Hello, world!'
  }
};

These objects are called “React elements”. You can think of them as descriptions of what you want to see on the screen【屏幕】. React reads【读取】 these objects and uses them to construct【构建】 the DOM and keep it up to date【更新】.

We will explore【输出】 rendering React elements to the DOM in the next section.

Tip:

We recommend【推荐】 using the “Babel” language definition for your editor of choice so that both ES6 and JSX code is properly【合理】 highlighted【高亮】. This website uses the Oceanic Next color scheme which is compatible【兼容】 with it.
