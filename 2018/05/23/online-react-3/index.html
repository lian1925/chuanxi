rendering【渲染】elements【元素】

作者： Lian

日期： 2018年05月23日 17:05

Elements are the smallest【最小】 building blocks【块】 of React apps.

An element【元素】 describes what you want to see【看到】 on the screen【屏幕】:

const element = <h1>Hello, world</h1>;

Unlike browser【浏览器】 DOM elements, React elements are plain【纯净】 objects【对象】, and are cheap【廉价】 to create【创建】. React DOM takes care of updating【更新】 the DOM to match the React elements.

Note【注意】:

One might confuse【迷惑】 elements with a more widely known concept【概念】 of “components”【组件】. We will introduce【介绍】 components【组件】 in the next section. Elements【元素】 are what components are “made of”, and we encourage【推荐】 you to read this section before jumping【跳】 ahead.

Rendering【渲染】 an Element【元素】 into the DOM

Let’s say there is a <div> somewhere in your HTML file:

<div id="root"></div>

We call this a “root” DOM node【节点】 because everything inside it will be managed【管理】 by React DOM.

Applications built with just React usually【通常】 have a single【单一】 root【根】 DOM node【节点】. If you are integrating【整合】 React into an existing【存在】 app, you may have as many isolated【孤立】 root DOM nodes as you like.

To render【渲染】 a React element into a root DOM node, pass both to ReactDOM.render():

embed:rendering-elements/render-an-element.js

It displays “Hello, world” on the page.

Updating【更新】 the Rendered Element

React elements are 【不可变】immutable. Once you create an element, you can’t change its children【子块】 or attributes【属性】. An element is like a single frame in a movie: it represents the UI at a certain point in time.

With our knowledge so far【目前】, the only way to update the UI is to create a new【新】 element, and pass it to ReactDOM.render().

Consider【思考】 this ticking clock example:

embed:rendering-elements/update-rendered-element.js

It calls ReactDOM.render() every second from a setInterval() callback.

Note:

In practice【实际】, most React apps only call ReactDOM.render() once【一次】. In the next sections we will learn how such code gets encapsulated into stateful components.

We recommend that you don’t skip【跳过】 topics【主题】 because they build on each other.

React Only Updates【更新】 What’s Necessary

React DOM compares【比较】 the element【元素】 and its children【子块】 to the previous【之前】 one, and only applies the DOM updates【更新】 necessary to bring the DOM to the desired【要求】 state.

You can verify【校验】 by inspecting【检查】 the last example with the browser【浏览器】 tools【工具】:

Even though【尽管】 we create an element describing【描述】 the whole UI tree on every tick, only the text node whose contents【内容】 has changed gets updated by React DOM.

In our experience【经验】, thinking about how the UI should look at any given moment【时刻】 rather than how to change it over time eliminates【消除】 a whole class of bugs.
