Package Versioning【版本】

作者： Lian

日期： 2018年05月26日 12:05

When someone references【参考】 your package from the web UI they will get a specific【特定】 version【版本】.

If you have any tags【标签】, PXT will pick the one with the highest Semantic Version precedence (the biggest【最大】 version
number). Thus【因此】, it’s good to have tags【标签】 like v0.0.0, v0.1.7 etc.

If there are no tags, PXT will pick the latest【最新】 commit from the default【默认】 branch
(usually master).

In both cases, the specific【指定】 version is hard-coded into the user’s package. The version【版本】 tag is set in the pxt.json for the package:

{
    "name": "neopixel",
    "version": "0.3.10",
    ...
}

You can use pxt bump to bump version of a package. It will git pull, update【更新】 the patch version【版本】 level (but will ask you for an override), create a git tag and push.

Package updates【更新】

To update, the user【用户】 has to take explicit【明确】 action【行为】 (currently【当前】, remove and re-add the package using the editor).

Forking packages【包】

Be aware that if you just fork an existing【存在】 package, the fork won’t show up in search.
You have to create【创建】 a new repo.
