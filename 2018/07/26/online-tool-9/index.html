eslint 代码格式检查 9

作者： Lian

日期： 2018年07月26日 19:07

1 知识点

eslint ，用于自动检查代码格式，保存代码风格的一致性。

1.1 安装

npm install -g eslint

1.2 实践

1、在项目的根目录创建配置文件

eslint --init

2、配置相关规则，修改文件 .eslintrc

{
    "rules": {
        "semi": ["error", "always"],
        "quotes": ["error", "double"]
    }
}

3、执行检查

eslint yourfile.js

1.3 eslint-config-airbnb

airbnb定义了一整套规范的react的eslint规则，是当前最受欢迎的开源项目。本节介绍如何在项目中引入。

1、查看相关依赖

npm info "eslint-config-airbnb@latest" peerDependencies

2、得到结果

{ eslint: '^4.19.1',
  'eslint-plugin-import': '^2.12.0',
  'eslint-plugin-jsx-a11y': '^6.0.3',
  'eslint-plugin-react': '^7.9.1' }

3、将依赖添加入package.json

4、执行安装

npm i
