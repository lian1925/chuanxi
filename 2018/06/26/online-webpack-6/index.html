消除死区代码

你的应用可以想象成一棵树。
应用实际用到的源码和库，代表绿色的枝叶。
应用并未使用的源码和库，也称为死区代码，代表枯黄的枝叶，为了优化代码，我们摇晃，去除无用的枝叶。

有3种建议：
使用 es6 的 import 和 export 语法。

为package.json添加 ‘sideEffects’ 属性。

{
  "name": "your-project",
  "sideEffects": false
}

使用插件UglifyJSPlugin
修改文件webpack.config.js

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
- mode: "development"
+ mode: "production"
};
